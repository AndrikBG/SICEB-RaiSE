# Implementation — ADD specs and phase guides (repository)

This folder holds **versioned** implementation material for SICEB.

**Normative / planning (use these to build and align scope):**

- [`implementationPlan.md`](implementationPlan.md) — central index: stack, dependencies, transition criteria.
- [`requeriments1.md`](requeriments1.md) … [`requeriments3.md`](requeriments3.md) — ADD architectural specifications for iterations 1–3.
- [`phase1-foundation.md`](phase1-foundation.md) through [`phase7-reporting.md`](phase7-reporting.md) — phase task checklists.

**Project memory and history (chronicle; not a substitute for specs above):**

- [`progress.md`](progress.md) — running log: phase status, decisions, blockers, session notes. Treat as **audit trail**, not as the single source of truth for requirements.
- [`summary/`](summary/) — **archived** phase-completion notes and file-level change digests from past work. Useful for onboarding and blame archaeology.
