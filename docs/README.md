3. **Iteration plan** — [`ADD/Design/IterationPlan.md`](ADD/Design/IterationPlan.md) (ordered delivery and driver traceability).
4. **Architecture** — [`ADD/Design/Architecture.md`](ADD/Design/Architecture.md) (large reference; use the document’s index and jump to the section you need).
5. **Drivers and constraints** — [`ADD/drivers/ArchitecturalDrivers.md`](ADD/drivers/ArchitecturalDrivers.md) and [`Requirements/Constraints.md`](Requirements/Constraints.md).
6. **Implementation** — [`ADD/implementation/implementationPlan.md`](ADD/implementation/implementationPlan.md) and phase guides `phase1-foundation.md` … `phase7-reporting.md`; current status in [`ADD/implementation/progress.md`](ADD/implementation/progress.md) (project log, not a normative spec).
