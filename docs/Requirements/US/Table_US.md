##   USER STORIES

| ID         | User Story (As a... I want... so that...)                                                                                                                                                                                                         | Priority   |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------- |
| **US-001** | As an **Administrator**, I want to create user accounts with role-based permissions so that staff can access the system according to their responsibilities                                                                                       | **HIGH**   |
| **US-002** | As a **system user**, I want to log in with my credentials to access the system securely                                                                                                                                                          | **HIGH**   |
| **US-003** | As an **Administrator**, I want to assign different permission levels (Director, Admin, Manager, Physician, Resident, etc.) so that users only see information relevant to their role                                                             | **HIGH**   |
| **US-004** | As a **General Administrator**, I want to see the complete inventory of ALL services to have total visibility of clinical resources                                                                                                               | **HIGH**   |
| **US-005** | As a **Service Manager**, I want to see ONLY the inventory of my service to manage the resources of my area effectively                                                                                                                           | **HIGH**   |
| **US-006** | As a **Service Manager**, I want to see automatic alerts when inventory items are running low to request restocking before running out of stock                                                                                                   | **MEDIUM** |
| **US-007** | As a **Service Manager**, I want to see alerts for items close to expiration to use them before they expire and minimize waste                                                                                                                    | **LOW**    |
| **US-008** | As **Lab Personnel**, I want to receive alerts when reagents are outside the safe temperature range to take corrective action immediately                                                                                                         | **MEDIUM** |
| **US-009** | As a **Service Manager**, I want to send a digital supply request so that the process is faster and traceable                                                                                                                                     | **MEDIUM** |
| **US-010** | As a **General Administrator**, I want to receive automatic notifications when supply requests are sent to review them promptly                                                                                                                   | **MEDIUM** |
| **US-011** | As a **General Administrator**, I want to approve or reject supply requests with justification so that decisions are documented                                                                                                                   | **MEDIUM** |
| **US-012** | As a **Service Manager**, I want to receive automatic notifications when my request is approved or rejected to know the status without manual follow-up                                                                                           | **LOW**    |
| **US-013** | As a **General Administrator** and **Service Manager**, we want to complete the material delivery cycle (Admin records delivery → Manager confirms receipt → Inventory updates) to close the request flow with complete traceability              | **MEDIUM** |
| **US-015** | As a **Service Manager** or **Resident**, I want to send a digital workshop/training request so that the approval process is formalized                                                                                                           | **LOW**    |
| **US-016** | As a **General Administrator**, I want to approve or reject workshop requests so that training activities are controlled                                                                                                                          | **LOW**    |
| **US-017** | As a **General Administrator**, I want to see the history of all workshop requests to track training activity                                                                                                                                     | **LOW**    |
| **US-018** | As a **Service Manager**, I want to register attendance of completed workshops so that resident participation is documented                                                                                                                       | **LOW**    |
| **US-019** | As **Reception Personnel**, I want to register new patients with their demographic information so they can receive medical attention                                                                                                              | **HIGH**   |
| **US-020** | As **Reception Personnel**, I want to classify patients by type (Student/Worker/External) AND have the system automatically apply the corresponding discount (30% Students, 20% Workers, 0% External) so that billing is consistent and automatic | **HIGH**   |
| **US-022** | As **Reception Personnel**, I want to register a minor patient with their guardian's information to align with legal requirements                                                                                                                 | **MEDIUM** |
| **US-023** | As **Reception Personnel**, I want the system to validate that a guardian is present for patients under 17 years old to enforce clinic policies                                                                                                   | **HIGH**   |
| **US-024** | As a **Physician** or **Service Manager**, I want to create a new medical record when I register a patient to start their medical history                                                                                                         | **HIGH**   |
| **US-025** | As a **Physician**, I want to add a new consultation entry to a patient's existing record to document their care                                                                                                                                  | **HIGH**   |
| **US-026** | As a **Physician**, I want consultation entries to be immutable (not editable) so that medical records maintain integrity for audits                                                                                                              | **HIGH**   |
| **US-027** | As a **Physician**, I want to see the comprehensive medical history of a patient from all services to have full context for treatment decisions                                                                                                   | **HIGH**   |
| **US-028** | As a **Physician**, I want to search for patients by name, ID, or student/employee number to quickly access their records                                                                                                                         | **MEDIUM** |
| **US-029** | As a **Physician**, I want to attach files (PDFs, images) to a patient's record so that external documents are consolidated                                                                                                                       | **MEDIUM** |
| **US-030** | As a **Physician**, I want to register vital signs during a consultation so that patient health trends can be tracked                                                                                                                             | **MEDIUM** |
| **US-031** | As a **Physician**, I want to prescribe medications during a consultation so that the prescription is recorded and available for pharmacy                                                                                                         | **HIGH**   |
| **US-032** | As **Pharmacy Personnel**, I want to see a patient's prescriptions to dispense the correct medications                                                                                                                                            | **HIGH**   |
| **US-033** | As **Pharmacy Personnel**, I want the system to validate that a prescription exists before dispensing to not dispense without a medical order                                                                                                     | **HIGH**   |
| **US-034** | As **Pharmacy Personnel**, I want the system to verify inventory before dispensing to know if medications are available                                                                                                                           | **HIGH**   |
| **US-035** | As **Pharmacy Personnel**, I want to register when I dispense controlled medications for complete traceability                                                                                                                                    | **HIGH**   |
| **US-036** | As **Pharmacy Personnel**, I want to register the dispensation with who prescribed, who dispensed, and to whom for a complete audit trail                                                                                                         | **MEDIUM** |
| **US-037** | As **Pharmacy Personnel**, I want to charge for medications separately from the consultation so that accounting is exact                                                                                                                          | **MEDIUM** |
| **US-038** | As a **Physician**, I want to request laboratory studies during a consultation to order diagnostic tests                                                                                                                                          | **HIGH**   |
| **US-039** | As **Reception Personnel**, I want to charge for laboratory studies BEFORE performing them to ensure payment in advance                                                                                                                           | **MEDIUM** |
| **US-040** | As **Lab Personnel**, I want to see pending study requests to know which tests to process                                                                                                                                                         | **HIGH**   |
| **US-041** | As **Lab Personnel**, I want to enter study results in text format so that physicians can review them                                                                                                                                             | **HIGH**   |
| **US-042** | As a **Physician**, I want to see laboratory results in the patient's medical record to interpret them within context                                                                                                                             | **HIGH**   |
| **US-043** | As **Lab Personnel**, I want to manage reagent inventory with expiration dates and temperature monitoring so that test quality is maintained                                                                                                      | **MEDIUM** |
| **US-044** | As **Reception Personnel**, I want to register payments for consultations, medications, and laboratory studies so that all income is tracked                                                                                                      | **HIGH**   |
| **US-045** | As **Reception Personnel**, I want the system to generate simple receipts and CFDI invoices so that patients have proof of payment compliant with tax regulations                                                                                 | **MEDIUM** |
| **US-046** | As a **Director** or **Administrator**, I want to generate financial reports showing income by service, concept, and patient type to analyze profitability                                                                                        | **MEDIUM** |
| **US-047** | As a **Director** or **Administrator**, I want to generate reports showing expenses by service to control costs                                                                                                                                   | **MEDIUM** |
| **US-048** | As a **Director**, I want to see profitability reports (income - expenses) by service to make strategic decisions                                                                                                                                 | **MEDIUM** |
| **US-049** | As an **Administrator**, I want to register medical personnel (attending and residents R1-R4) so that staff is in the system                                                                                                                      | **MEDIUM** |
| **US-050** | As the **system**, I want to automatically validate that residents can only perform actions allowed for their level (R1-R4) to enforce clinic policies                                                                                            | **HIGH**   |
| **US-051** | As the **system**, I want to block R1, R2, and R3 Residents from prescribing controlled medications to enforce security policies                                                                                                                  | **HIGH**   |
| **US-052** | As a **Physician**, I want to register which supplies I used during a consultation so that inventory updates and costs are tracked                                                                                                                | **MEDIUM** |
| **US-053** | As **Reception Personnel**, I want to schedule medical appointments for patients specifying date, time, physician, and consultation type so that the consultation flow is organized                                                               | **MEDIUM** |
| **US-054** | As a **Physician**, I want to see my appointment agenda for the day with patient information to know which patients I will attend and prepare appropriately                                                                                       | **MEDIUM** |
| **US-055** | As **Reception Personnel**, I want to cancel or reschedule medical appointments documenting the reason to manage patient changes                                                                                                                  | **MEDIUM** |
| **US-056** | As **Administrative Personnel**, I want to register signed informed consents by attaching a scanned document to comply with NOM-024-SSA3-2012                                                                                                     | **MEDIUM** |
| **US-057** | As a **Physician**, I want to mark in the record that a procedure requires specific consent to document legal compliance                                                                                                                          | **MEDIUM** |
| **US-058** | As a **Service Manager**, I want to register discarded materials with a reason (expired, damaged) to document losses                                                                                                                              | **LOW**    |
| **US-059** | As a **Pediatrician**, I want to register growth data and vaccination schedule in pediatric records for proper follow-up                                                                                                                          | **MEDIUM** |
| **US-060** | As a **Gynecologist**, I want to register pregnancy data (LMP, weeks of gestation, EDD) for prenatal follow-up                                                                                                                                    | **MEDIUM** |
| **US-061** | As **Emergency Personnel**, I want to assign triage level (Red/Yellow/Green) to prioritize care according to severity                                                                                                                             | **MEDIUM** |
| **US-062** | As a **Patient**, I want to request access to my complete medical record (ARCO right) to exercise my rights                                                                                                                                       | **MEDIUM** |
| **US-063** | As **Administrative Personnel**, I want to process ARCO requests within the legal timeframe (20 days) to comply with LFPDPPP                                                                                                                      | **MEDIUM** |
| **US-064** | As an **Administrator**, I want to configure tariffs by medical service specifying base price so that the system calculates charges correctly                                                                                                     | **HIGH**   |
| **US-065** | As an **Administrator**, I want to assign a supervisor physician to R1/R2 residents to comply with mandatory supervision policies                                                                                                                 | **MEDIUM** |
| **US-066** | As the **system**, I want to register in an audit log who accessed which record and when to comply with LFPDPPP traceability                                                                                                                      | **HIGH**   |
| **US-067** | As a **Director**, I want to see an executive dashboard with key metrics to monitor clinic performance                                                                                                                                            | **LOW**    |
| **US-068** | As an **Administrator**, I want to generate a report of supply requests by service and period to analyze consumption patterns                                                                                                                     | **LOW**    |
| **US-070** | As the **system**, I want to send appointment reminders 24h before to reduce absenteeism                                                                                                                                                          | **LOW**    |
| **US-071** | As a **General Administrator**, I want to register a new Branch (Sucursal) with its details (name, address) so that the system can manage operations in that location                                                                             | **HIGH**   |
| **US-072** | As a **General Administrator**, I want to deactivate a Branch so that it no longer allows new operations but preserves history                                                                                                                    | **LOW**    |
| **US-073** | As an **Administrator**, I want to assign one or multiple authorized branches to a User so that they can operate in those specific locations                                                                                                      | **MEDIUM** |
| **US-074** | As a **Multi-Branch User**, I want to select my "Active Branch" upon login so that the system shows me the correct inventory and patients for that location                                                                                       | **HIGH**   |
| **US-075** | As a **General Administrator**, I want to see a consolidated dashboard of all branches to compare performance metrics across the network                                                                                                          | **MEDIUM** |
| **US-076** | As a **Branch User**, I want to continue registering consultations when internet is lost so that operations are not interrupted, with automatic sync when restored                                                                                | **HIGH**   |
