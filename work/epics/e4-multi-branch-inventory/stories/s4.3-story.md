# Story: Tariff Configuration

## User Story
As a General Administrator,
I want to configure service tariffs with effective dates per branch,
so that pricing changes take effect on a scheduled date without altering historical charges.

## Acceptance Criteria

### Scenario: Create tariff for a service in a branch
```gherkin
Given I have tariff:manage permission
  And branch "Central" exists with service "Consulta General"
When I create a tariff with basePrice 150.0000 and effectiveFrom "2026-05-01"
Then the tariff is persisted with DECIMAL(19,4) precision
  And the tariff is associated to the branch and service
```

### Scenario: Resolve active tariff
```gherkin
Given service "Consulta General" in branch "Central" has tariffs:
  | basePrice | effectiveFrom       |
  | 100.0000  | 2026-01-01T00:00:00 |
  | 150.0000  | 2026-04-01T00:00:00 |
  | 200.0000  | 2026-07-01T00:00:00 |
When I resolve the active tariff on 2026-05-15
Then the active tariff has basePrice 150.0000
  And effectiveFrom 2026-04-01
```

### Scenario: Update tariff creates new effective-date entry
```gherkin
Given tariff T1 exists for "Consulta General" in "Central" at 150.0000
When I update with basePrice 175.0000 and effectiveFrom "2026-06-01"
Then a new tariff record is created (T1 unchanged — immutable)
  And the active tariff before 2026-06-01 remains 150.0000
```

### Scenario: Prevent duplicate effective dates
```gherkin
Given a tariff exists for service S in branch B with effectiveFrom "2026-05-01"
When I create another tariff for the same service/branch with effectiveFrom "2026-05-01"
Then the request is rejected with a conflict error
```

### Scenario: RLS enforces branch isolation
```gherkin
Given I am authenticated in branch "Central"
When I query tariffs
Then I only see tariffs for branch "Central"
  And tariffs from other branches are not visible
```

### Scenario: Free service tariff
```gherkin
Given I have tariff:manage permission
When I create a tariff with basePrice 0.0000
Then the tariff is accepted (free services are valid)
```

## Examples (Specification by Example)

| Input | Action | Expected Output |
|-------|--------|-----------------|
| basePrice=150.0000, effectiveFrom=2026-05-01 | CreateTariff | Tariff persisted, 201 Created |
| serviceId=X, branchId=Y | GetActiveTariff | Tariff with MAX(effectiveFrom) <= NOW() |
| basePrice=-10.00 | CreateTariff | 400 Bad Request — negative price |
| duplicate (service, branch, effectiveFrom) | CreateTariff | 409 Conflict |
| search "Consulta" | SearchTariffs | Tariffs matching service name substring |

## Notes
- Design decision D-042: temporal effective-date pattern with immutable history
- All prices DECIMAL(19,4) per CRN-42
- RLS on service_tariffs per D-043 + T4.5.2
- Permissions: tariff:manage (create/update), tariff:read (view) — seeded in migration
- UNIQUE(service_id, branch_id, effective_from) prevents overlapping dates
- UpdateTariff semantics: effectiveFrom must be future, creates new record (never mutates)
