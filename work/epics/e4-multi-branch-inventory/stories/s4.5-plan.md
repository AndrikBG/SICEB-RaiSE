## Task List

cd apps/backend && mvn test -pl . -Dtest=PartitionPruningTest -Dspring.profiles.active=integration

**Dependencies:** None

### T2: k6 HTTP + WebSocket load scripts (M)

**Description:** Create k6 load test scripts that:
1. Authenticate via `POST /api/auth/login`, obtain JWT
2. HTTP scenarios: per-branch inventory query (`GET /api/inventory`) and patient search (`GET /api/clinical/patients/search`) at 3, 9, 15 branch counts
3. WebSocket scenario: 150 concurrent STOMP connections with JWT auth, subscribe to branch topics, measure message delivery latency
4. Thresholds: <10% p95 degradation, <2s WebSocket delivery, <1s patient search

**Files:**
- Create: `tests/scalability/http-load.js` (inventory queries + patient search)
- Create: `tests/scalability/ws-load.js` (WebSocket/STOMP connections)
- Create: `tests/scalability/helpers.js` (auth, STOMP frame builders, branch data)
- Create: `tests/scalability/run-scalability.sh` (orchestrator: seed → k6 HTTP → k6 WS → collect results)

**TDD cycle:**
- RED: Run k6 scripts against Docker Compose with 3 branches — scripts fail (no infrastructure, no seeding)
- GREEN: Wire up `run-scalability.sh` to: start Docker Compose, apply seed SQL, run k6 with `--out json`, capture results. Scripts pass with 3 branches.
- REFACTOR: Add threshold assertions in k6 config, parameterize VU count

**Verification:**
```bash
# Requires Docker Compose running + k6 installed
./tests/scalability/run-scalability.sh --branches 3
```

**AC coverage:** AC2, AC3 (query degradation), AC4 (WebSocket), AC6 (patient search)
**Dependencies:** T1 (seed SQL)

**AC coverage:** AC7 (scalability report)

## Execution Order

```
T1 (seed + pruning test) → T2 (k6 scripts) → T3 (execute + report)
```

Sequential — each task depends on the previous. Risk-first: T1 validates the foundational assumption (partitioning works) before building load scripts on top.

## Risks & Mitigations

| # | Risk | Mitigation |
|---|------|------------|
| 1 | k6 not installed on dev machine | `run-scalability.sh` checks for k6 binary, prints install instructions |
| 2 | Docker Compose build slow (first time) | Pre-build: `docker compose build` before running tests |
| 3 | 150 WebSocket connections may hit OS file descriptor limits | Document `ulimit -n` requirement in run script |
| 4 | Patient search table may lack >50K records in seed | Seed script generates 50K+ patient records explicitly |
