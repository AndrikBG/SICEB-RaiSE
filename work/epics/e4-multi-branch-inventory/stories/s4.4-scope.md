# Scope — S4.4: PWA Views

**Story:** S4.4 | **Size:** M | **Date:** 2026-04-13
**Epic:** E4 — Multi-Branch Operations and Inventory
**Branch:** `story/s4.4/pwa-views`
**Dependencies:** S4.1 (done), S4.2 (done), S4.3 (done)

## In Scope

- BranchManagementView: branch CRUD UI (register, list, deactivate, onboarding progress)
- InventoryDashboardView: full inventory grid for admin (filter, sort, search, paginate, export, color-coded status, real-time updates)
- ServiceInventoryView: service-scoped inventory for service managers (pre-filtered, unauthorized items never rendered)
- TariffConfigurationView: tariff catalog with create, update, search, price validation
- BranchSelectionView update: mid-session context switching (confirm → POST → JWT → clear cache → resubscribe WS → reload)
- React Router: /branches, /inventory, /inventory/service, /tariffs — permission-gated
- API clients: branch-api.ts, tariff-api.ts (inventory-api.ts already exists)

## Out of Scope

- Offline sync for inventory (Phase 6)
- PWA offline caching of branch/tariff data
- Mobile-specific responsive layouts (desktop-first, responsive is bonus)
- Unit testing of React components (integration testing via browser in T3)
- Navigation sidebar/menu redesign — add links to existing structure

## Done Criteria

- [ ] BranchManagementView renders for users with `branch:manage`, hidden otherwise
- [ ] InventoryDashboardView shows paginated, filtered, color-coded grid with real-time WebSocket updates
- [ ] ServiceInventoryView shows only assigned-service inventory, never leaks other services
- [ ] TariffConfigurationView shows catalog, creates/updates tariffs, validates prices
- [ ] Branch switching clears cache, resubscribes WebSocket, reloads view
- [ ] Routes /branches, /inventory, /inventory/service, /tariffs are permission-gated
- [ ] All views interact with real backend (Docker Compose verification)
