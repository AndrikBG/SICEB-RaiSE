## Key Requirements from US (summary for quick reference)

### US-004 — Complete Inventory View (Admin)
- Items: SKU, name, category, service, current_stock, min_threshold, expiration, status
- Grouped by service, filterable by category/status/service, sortable, searchable
- Detail panel on item selection
- Export to Excel
- Pagination at 50 items/page
- Color-coded status: OK=white, LOW_STOCK=yellow, EXPIRING_SOON=orange, EXPIRED=red

### US-005 — Service-Scoped Inventory View (Service Manager)
- Same fields, pre-filtered to user's service
- Items from other services never visible (RLS enforced)
- 403 on unauthorized direct access + audit log
- Export scoped to own service only

## Design Decisions (from docs/ADD)

| # | Decision | Pattern |
|---|----------|---------|
| D-036 | CQRS append-only `InventoryDeltaStore` + PG trigger materialization | Same as ClinicalEventStore (Phase 2) |
| D-037 | WebSocket via Spring STOMP + `pg_notify` bridge | Tenant-scoped channels |
| D-040 | PostgreSQL declarative partitioning by `branch_id` | Partition pruning for scalability |
| D-041 | `SELECT ... FOR UPDATE` + timestamp-ordered delta application | Concurrency serialization |
| D-043 | RLS on inventory + WebSocket subscription auth | Three-layer defense-in-depth |
