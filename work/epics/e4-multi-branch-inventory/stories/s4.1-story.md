# S4.1 — Branch Management and Context Switching

## User Story (Connextra)

**As a** General Administrator,
**I want to** register new branches, manage their lifecycle, and switch between branches mid-session without logging out,
**so that** the clinic network can onboard new locations efficiently and staff can work across branches seamlessly.

## Acceptance Criteria (Gherkin)

### Branch Registration

```gherkin
Scenario: Register a new branch
  Given I am authenticated as General Administrator
  When I POST /api/branches with name "Clínica Norte" and address "Av. Norte 123"
  Then the response is 201 with a branchId (UUID)
  And the branch appears in GET /api/branches
  And an audit event is recorded

Scenario: Reject duplicate branch name
  Given a branch named "Clínica Norte" already exists
  When I POST /api/branches with name "Clínica Norte"
  Then the response is 409 RESOURCE_ALREADY_EXISTS

Scenario: Deactivate a branch
  Given branch "Clínica Norte" exists and is active
  When I POST /api/branches/{branchId}/deactivate
  Then the branch isActive becomes false
  And new operations for that branch are blocked
  And existing data is preserved
```

### Branch Onboarding

```gherkin
Scenario: 5-step onboarding completes after registration
  Given a new branch was just registered
  When the onboarding orchestrator runs
  Then steps execute in order: partitions, catalog, tariffs, inventory, mark complete
  And each step is recorded in branch_onboarding_status
  And the total process completes in under 1 hour (ESC-01)

Scenario: Onboarding is resumable on failure
  Given onboarding failed at step 3 (tariff copy)
  When the orchestrator is triggered again
  Then it resumes from step 3 (not step 1)
  And completed steps are not re-executed
```

### Branch Context Switching

```gherkin
Scenario: Switch branch mid-session (ESC-03)
  Given I am authenticated with branches [A, B] assigned
  And my active branch is A
  When I POST /api/session/branch with branchId=B
  Then I receive a new JWT with activeBranchId=B
  And the switch completes in under 3 seconds
  And subsequent queries are scoped to branch B via RLS

Scenario: Reject switch to unassigned branch
  Given I am only assigned to branch A
  When I POST /api/session/branch with branchId=C
  Then the response is 403

Scenario: Single-branch user auto-selects
  Given I am assigned to exactly one branch
  When I log in
  Then the branch is auto-selected without showing selection UI
```

## Specification by Example

| Input | Expected |
|-------|----------|
| Register branch with valid name/address | 201, branchId returned, onboarding starts |
| Register branch with empty name | 400 VALIDATION_FAILED |
| Register branch with duplicate name | 409 RESOURCE_ALREADY_EXISTS |
| Deactivate active branch | 200, isActive=false |
| Deactivate already inactive branch | 409 |
| Switch to assigned branch | 200, new JWT, <3s |
| Switch to unassigned branch | 403 |
| Get branches as admin | All branches (active + inactive) |
| Get branches as non-admin | Only assigned branches |
| Onboarding step fails and retries | Resumes from failed step |
