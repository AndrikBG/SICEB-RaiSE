## Key Requirements from US (summary for quick reference)

### US-071 — Branch Registration
- Fields: name (unique), address (street/city/state/postal), phone, email, operating hours, branch code (optional)
- Only General Administrator can register
- 5-step onboarding starts automatically after registration
- Onboarding progress visible in branch detail panel
- Onboarding < 1 hour (ESC-01)
- Edit existing branch details (name, address, phone, email, hours)
- Deactivation preserves history
- Audit event on registration

### US-074 — Active Branch Selection
- Select at login: list of authorized active branches
- Mid-session switch: new JWT, no re-auth, < 3 seconds (ESC-03)
- Data isolation: all views reload scoped to new branch
- WebSocket re-subscription to new branch channels
- Local cache cleared on switch
- Single-branch users auto-select (skip selection screen)
- Remember last active branch per user
- Deactivated branches hidden from selector
- No authorized branches → error, cannot proceed

## Implementation Notes

- `Branch` entity needs new fields from US-071: phone, email, opening_time, closing_time, branch_code
- US-074 "remember last branch" needs `last_active_branch_id` on users table or separate preference
- US-074 WebSocket re-subscription is documented here but actual WebSocket changes are S4.2
- Onboarding step 5 (WebSocket channel) is a no-op in S4.1 — real channels created in S4.2
