# Scope — S4.5: Scalability Validation

## In Scope

- Data seeding scripts: generate realistic inventory deltas, stock, tariffs for 3→15 branches
- k6 load test scripts for per-branch queries (partition-pruned) and cross-branch admin queries
- k6 WebSocket load test: 150 concurrent STOMP connections with JWT auth
- EXPLAIN ANALYZE validation of partition pruning on inventory_deltas and stock_current
- RLS scaling verification across 3, 9, 15 branch counts
- Patient search benchmark with >50K seeded records (carried from Phase 2, E2.8)
- Scalability report documenting baselines, results, and pass/fail against thresholds
- Performance baseline documentation for inventory queries and WebSocket latency

## Out of Scope

- Horizontal WebSocket scaling (single-instance sufficient per scope.md)
- Database sharding or cross-node distribution
- Production deployment configuration
- New feature code (this is validation-only)
- CI/CD integration of load tests (future concern)
- Offline sync performance (Phase 6)

## Done Criteria

- [ ] Data seeding generates consistent test data for 3, 9, and 15 branches
- [ ] Per-branch query latency degrades <10% from 3→15 branches
- [ ] Cross-branch admin query latency degrades <10% from 3→15 branches
- [ ] 150 concurrent WebSocket connections sustained with <2s message delivery
- [ ] EXPLAIN ANALYZE confirms partition pruning on partitioned tables
- [ ] Patient search returns in <1s with >50K records
- [ ] Scalability report committed with baselines, measurements, and conclusions
