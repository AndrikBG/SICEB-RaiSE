## What Went Well

- **Risk-first ordering paid off**: T1→T2 validated the riskiest unknowns (partitioning, triggers, pg_notify) before any domain code. When T7 found the epoch-day trigger bug, the fix was a single migration — no cascade.
- **JwtClaimsPrincipalMapper extraction**: Refactoring JWT claims parsing into a shared mapper during T5 eliminated duplication between JwtAuthenticationFilter and WebSocketSecurityInterceptor. Clean separation.
- **Full AC validation against live system**: Docker Compose + curl + psql caught 4 real defects that unit tests missed. The investment in T7 validation was worth it.
- **180 tests, all green**: High confidence in the implementation.

## What to Improve

- **T2 integration test coverage gap**: The EXPIRATION delta type was never tested in schema integration tests — only INCREMENT, DECREMENT, ADJUST were covered. The trigger date conversion bug survived until T7. **Fix: always test ALL delta types in trigger integration tests** (PAT-A-014).
- **@PreAuthorize SpEL assumption**: Used `@auth.check('a') or @auth.check('b')` without testing against real SpEL evaluation. `@WebMvcTest` mocks don't catch this. **Fix: use `hasAnyAuthority` for OR semantics** (PAT-A-013).
- **Insufficient stock error path**: Assumed PG trigger errors would surface as `DataIntegrityViolationException` — they're `JpaSystemException` because triggers fire at commit. **Fix: added GlobalExceptionHandler for JpaSystemException** (PAT-A-012).
- **RealtimeEventPublisher LISTEN connection**: Holds a dedicated JDBC connection that triggers HikariCP leak detection. Works but needs production hardening (separate DataSource or leak-detection exclusion).

## Heutagogical Checkpoint

1. **What did I learn?** JPA/Hibernate wraps commit-time PG trigger exceptions differently than statement-time ones. SpEL `or` with throwing methods short-circuits on the exception. Epoch day → date in PG requires arithmetic, not casting.

2. **What would I change?** Add EXPIRATION integration test in T2. Test controller auth with real SpEL evaluation, not mocked. These defects cost ~2h in T7 debugging.

3. **Framework improvements?** Pattern PAT-A-014 now codifies "test all delta types". The trigger-as-materialization pattern is validated and documented (PAT-A-015).

4. **What am I more capable of?** End-to-end CQRS with PG trigger materialization. Three-layer WebSocket auth (CONNECT/SUBSCRIBE/RLS). pg_notify → STOMP bridge.

## Patterns Added

- **PAT-A-012**: PG trigger exceptions → JpaSystemException (not DIVE) — handle in GlobalExceptionHandler
- **PAT-A-013**: `hasAnyAuthority()` for SpEL OR when `@auth.check` throws
- **PAT-A-014**: Integration tests must cover ALL delta/event types for PG triggers
- **PAT-A-015**: pg_notify → LISTEN → @Scheduled → SimpMessagingTemplate validated pattern
