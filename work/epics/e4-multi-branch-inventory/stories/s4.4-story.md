# User Story — S4.4: PWA Views

## Story

**As a** branch administrator or service manager,
**I want** to manage branches, inventory, and tariffs through the PWA,
**so that** I can operate my branch without requiring desktop software or manual coordination.

## Size: M

## Acceptance Criteria

### AC1: Branch Management View

```gherkin
Scenario: Admin registers a new branch
  Given the user has "branch:manage" permission
  When they navigate to /branches
  Then they see a list of branches with name, status, and onboarding progress
  And they can register a new branch with name and address
  And they can deactivate an existing branch

Scenario: Non-admin cannot access branch management
  Given the user does NOT have "branch:manage" permission
  When they navigate to /branches
  Then the view is not rendered (RequireAnyPermission)
```

### AC2: Inventory Dashboard View (General Administrator)

```gherkin
Scenario: Admin views full inventory grid
  Given the user has "inventory:read" permission
  When they navigate to /inventory
  Then they see a paginated grid (50 items/page) with all branch inventory
  And items are color-coded by status: OK=white, LOW_STOCK=yellow, EXPIRING_SOON=orange, EXPIRED=red
  And they can filter by category, status, service, and search by name/SKU
  And they can export to Excel

Scenario: Real-time updates appear
  Given the user is on /inventory
  When another user increments stock for an item
  Then the grid updates in real-time via WebSocket (< 2 seconds)
```

### AC3: Service Inventory View (Service Manager)

```gherkin
Scenario: Service manager sees only their service's inventory
  Given the user has "inventory:read_service" permission
  When they navigate to /inventory/service
  Then they see only items belonging to their assigned service
  And items from other services are never rendered
```

### AC4: Tariff Configuration View

```gherkin
Scenario: Admin manages tariffs
  Given the user has "tariff:manage" permission
  When they navigate to /tariffs
  Then they see a catalog of services with current price and effective date
  And they can create a new tariff with future effective date
  And they can update price with a new effective date (immutable history)
  And price validation rejects negative values
  And they can search across the catalog
```

### AC5: Branch Selection — Context Switching

```gherkin
Scenario: User switches branch mid-session
  Given the user is authenticated and on any view
  When they switch branches via BranchSelectionView
  Then the app: (1) confirms unsaved work, (2) POSTs /session/branch,
       (3) stores new JWT, (4) clears branch-scoped cache,
       (5) resubscribes WebSocket, (6) reloads dashboard
```

### AC6: React Router Integration

```gherkin
Scenario: Routes exist and are permission-gated
  Given the app is loaded
  Then /branches is protected by "branch:manage"
  And /inventory is protected by "inventory:read"
  And /inventory/service is protected by "inventory:read_service"
  And /tariffs is protected by "tariff:manage"
```
